DIGITAL LOGIC · VIVADO · VERILOG

Traffic Controller Enhancement

Pedestrian request input and waiting-state feedback

차량 감지에만 반응하던 5-state 교통신호 제어기에 보행자 요청 입력 btn_pedestrian과 대기 안내 출력 led_wait를 추가한 개인 디지털논리회로 프로젝트입니다. 기존 FSM 상태 수와 신호 주기는 유지하면서 입력 조건과 출력 논리를 확장했습니다.

01 / OVERVIEW

Extend the interface, preserve the FSM.

기준 시스템은 Main Highway가 기본 Green을 유지하고 Country Road의 차량 센서가 감지될 때만 신호 주기를 시작했습니다. 개선안은 보행자 버튼도 동일한 요청으로 받아들이고, 요청이 처리되는 동안 LED로 대기 상태를 알려줍니다.

BASE FSM5 states · S0–S4
NEW INPUTbtn_pedestrian
NEW OUTPUTled_wait
STATE COUNTUnchanged

02 / PROBLEM

A vehicle-only request path.

기존 controller에는 Country Road 차량 센서 외에 신호 전환을 요청할 방법이 없어 보행자가 접근해도 Main Highway의 Green이 유지됩니다. 또한 요청이 접수됐는지 사용자에게 보여주는 출력이 없어 입력과 실제 신호 전환 사이의 대기 상태를 알 수 없었습니다.

SAFETY GAP

No pedestrian trigger

보행자 입력을 차량 센서와 같은 우선순위의 요청으로 통합할 논리가 필요했습니다.

UX GAP

No acknowledgement

요청 직후부터 한 주기가 끝날 때까지 대기 상태를 표시할 출력이 필요했습니다.

03 / WORKFLOW

Specification to waveform.

  1. Read the baseline

    Main/Country 신호와 S0–S4 전이 조건, 색상 인코딩을 기준 사양으로 정리했습니다.

  2. Define new I/O

    btn_pedestrian 입력과 led_wait 출력을 port와 testbench에 반영했습니다.

  3. Reuse the state cycle

    car_country || btn_pedestrian 조건으로 동일한 S0→S1 경로를 사용해 새 상태를 만들지 않았습니다.

  4. Verify waveforms

    차량 진입, 보행자 버튼, LED 점등·소등을 시간축에서 확인했습니다.

04 / IMPLEMENTATION

Minimal RTL change, explicit feedback.

보행자 요청을 별도 state sequence로 분리하지 않고 차량 센서와 OR 조건으로 결합했습니다. 요청이 들어오면 S1부터 S4까지 led_wait를 1로 유지하고, 주기가 끝나 S0로 복귀할 때 소등하도록 출력 논리를 배치했습니다.

State machine

기존 5-state 흐름을 유지한 개선 구조입니다.

Pedestrian request

버튼 입력이 차량 감지와 같은 신호 주기를 시작함을 확인했습니다.

Waiting LED

S1–S4 동안 점등되고 S0 복귀 시 소등되는 출력입니다.

05 / RESULTS

Verified state behavior.

5

FSM states retained

2 → 1

Vehicle or pedestrian request path

S1–S4

Waiting LED asserted

  • 차량 입력이 high일 때 S0 → S1 → S2 → S3 → S4 순서로 전이하는 기존 동작을 확인했습니다.
  • 보행자 버튼 입력도 같은 state cycle을 시작하며 별도 상태 추가 없이 동작했습니다.
  • 요청 처리 중 led_wait가 유지되고 S0 복귀 시 소등되는 waveform을 보고서에 기록했습니다.

06 / CONTRIBUTION

My contribution.

  • 차량 중심 controller의 한계를 정의하고 보행자 버튼·대기 LED 요구사항을 직접 설계했습니다.
  • 기존 5-state FSM을 분석해 새로운 상태 없이 요청 조건과 출력만 확장했습니다.
  • Verilog RTL과 testbench 입력을 수정하고 Vivado waveform으로 차량·보행자·LED 동작을 검증했습니다.
  • 상태표, state diagram, 코드 설명, simulation 해석을 발표자료와 결과 보고서로 정리했습니다.

07 / LIMITS

Limitations & lessons.

  • 검증은 Vivado simulation waveform 수준이며 FPGA board 실장이나 physical timing closure 결과는 공개 자료에 없습니다.
  • 저장소에는 독립 .v 또는 testbench 파일이 없고, README 코드 블록과 PDF 스크린샷이 구현 근거입니다.
  • 현재 로직은 입력 debounce, 장시간 button hold, 동시 요청 우선순위 같은 실제 입력 장치의 edge case를 다루지 않습니다.
  • 새 상태를 추가하지 않고 기존 FSM을 재사용하면 변경 범위를 줄이면서도 사용자 피드백을 개선할 수 있음을 확인했습니다.

DIGITAL DESIGN · VERILOG FSM

Digital Logic

Traffic Signal Controller Enhancement

기존 교통신호 제어 FSM에 보행자 요청 입력과 대기 LED 피드백을 추가하고, 상태를 늘리지 않은 채 기존 전환 시퀀스를 재사용한 설계 프로젝트입니다.

PERIOD
2026-1
STATUS
Completed
ROLE
Logic design & simulation
TOOLS
Vivado, Verilog, FSM simulation

01 / OVERVIEW

Pedestrian input without extra states.

기존 차량 감지 기반 FSM에 보행자 요청 기능을 추가하면서 상태 수와 기본 신호 순서는 유지하는 것이 설계 방향이었습니다. 보행자 버튼 요청은 차량 입력과 OR 조건으로 결합해 같은 신호 전환 시퀀스를 사용하도록 구성했습니다.

Feedback behavior: 요청이 접수된 뒤 실제 보행 신호 전환을 기다리는 동안 대기 LED가 상태를 표시합니다.

02 / METHOD

Reuse the verified transition path.

  1. FSM review

    기존 상태와 차량 입력에 따른 전환 순서를 분석했습니다.

  2. Input integration

    보행자 요청을 기존 차량 입력과 OR 조건으로 결합했습니다.

  3. Simulation check

    상태 추가 없이 동일 시퀀스가 실행되고 대기 LED가 동작하는지 검증했습니다.

My contribution

  • 보행자 요청 입력과 대기 LED 출력 논리를 Verilog에 통합했습니다.
  • 기존 FSM 상태를 유지하는 조건에서 입력·출력 동작을 설계했습니다.
  • Vivado simulation으로 차량 입력과 보행자 요청의 전환 순서를 비교했습니다.

03 / RESULTS

Simulation-verified behavior.

차량 입력과 보행자 요청이 동일한 신호 전환 시퀀스를 실행하며, 보행자 요청 후 대기 상태가 LED로 표시되는 동작을 시뮬레이션으로 확인했습니다.

Pedestrian request

버튼 요청이 기존 전환 조건에 통합된 동작입니다.

Waiting LED feedback

요청 접수 후 신호 전환 전까지의 대기 표시를 검증했습니다.

Technical interpretation & lessons

  • 기존 상태 전환 경로를 재사용해 상태 수 증가를 피하고 검증 범위를 명확하게 유지했습니다.
  • 결과는 Vivado 시뮬레이션 검증이며 실제 보드의 입력 bounce나 물리 신호 timing 검증까지 포함하지 않습니다.
